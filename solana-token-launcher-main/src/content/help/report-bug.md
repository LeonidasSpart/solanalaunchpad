# Report a Bug

Help us improve ZRP by reporting bugs and issues.

## What to Report

### Types of Bugs

| Type | Description |
|------|-------------|
| **Technical** | Code errors, crashes |
| **UI/UX** | Display issues, usability |
| **Performance** | Slow loading, lag |
| **Security** | Vulnerabilities |
| **Data** | Incorrect information |

### Examples

| Bug | Type |
|-----|------|
| Token creation fails | Technical |
| Button not working | UI/UX |
| Page loading slowly | Performance |
| Wrong balance showing | Data |

## How to Report

### Option 1: Contact Form

1. Go to the [Contact page](/contact)
2. Subject: **"Bug Report: [Brief Description]"**
3. Include:
   - What happened
   - What you expected
   - Steps to reproduce
   - Screenshots

### Option 2: X (Twitter)

1. Tag @ZRP_AI
2. Include: **"Bug Report:"**
3. Describe the issue
4. Add screenshots

### Option 3: Discord

1. Join our [Discord](https://discord.gg/W4qS4xkbn)
2. Post in **#bug-reports**
3. Describe the issue
4. Tag the team

### Option 4: GitHub

1. Go to our [GitHub](https://github.com/LeonidasSpart/solanalaunchpad)
2. Open an issue
3. Label it: **"bug"**
4. Fill in the template

## What to Include

### Essential Information

| Item | Why |
|------|-----|
| **Description** | What happened |
| **Steps** | How to reproduce |
| **Expected** | What should have happened |
| **Actual** | What actually happened |
| **Screenshots** | Visual evidence |

### Additional Information

| Item | Why |
|------|-----|
| **Browser** | Chrome, Firefox, etc. |
| **Version** | Browser version |
| **Wallet** | Phantom, Solflare, etc. |
| **Network** | Devnet or Mainnet |
| **Device** | Desktop, mobile, etc. |

## What Happens Next

### Process

1. **Report received** — Team acknowledges
2. **Investigation** — Issue is reviewed
3. **Priority** — Issue is prioritized
4. **Fix** — Issue is fixed
5. **Release** — Fix is released

### Timeline

| Priority | Response Time | Fix Time |
|----------|---------------|----------|
| **Critical** | 1-4 hours | 1-2 days |
| **High** | 4-12 hours | 2-5 days |
| **Medium** | 12-24 hours | 1-2 weeks |
| **Low** | 24-48 hours | Next release |

## Severity Levels

### Critical

- 🚨 Platform down
- 🚨 Security breach
- 🚨 Data loss

### High

- ⚠️ Major feature broken
- ⚠️ Users affected
- ⚠️ Workaround needed

### Medium

- ℹ️ Minor feature broken
- ℹ️ UI issues
- ℹ️ Performance problems

### Low

- 📝 Cosmetic issues
- 📝 Documentation errors
- 📝 Minor bugs
