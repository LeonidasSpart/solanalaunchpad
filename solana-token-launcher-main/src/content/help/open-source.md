# Open Source

ZRP is fully open source. Anyone can view, audit, and contribute to the code.

## Why Open Source Matters

### Benefits of Open Source

| Benefit | Description |
|---------|-------------|
| **Transparency** | Code is visible to everyone |
| **Security** | Anyone can audit |
| **Trust** | No hidden logic |
| **Community** | Anyone can contribute |
| **Innovation** | Faster development |

## How It Works

### Code Availability

- ✅ **Public repository** on GitHub
- ✅ **Full source code** available
- ✅ **MIT License** — Free to use

### What You Can Do

1. **View** the code
2. **Audit** for security
3. **Fork** the project
4. **Contribute** improvements
5. **Build** your own version

## How to Contribute

### 1. Fork the Repository

1. Go to [GitHub](https://github.com/LeonidasSpart/solanalaunchpad)
2. Click **"Fork"**
3. Create your own copy

### 2. Make Changes

1. Clone your fork
2. Create a feature branch
3. Make your changes
4. Test thoroughly

### 3. Submit a Pull Request

1. Push your changes
2. Open a Pull Request
3. Describe your changes
4. Wait for review

## Contribution Guidelines

### What We're Looking For

- ✅ Bug fixes
- ✅ New features
- ✅ Documentation improvements
- ✅ Performance enhancements

### What to Avoid

- ❌ Breaking changes
- ❌ Unrelated changes
- ❌ Poor documentation
- ❌ Untested code

## Code Quality

### Standards

| Standard | Requirement |
|----------|-------------|
| **Language** | TypeScript |
| **Style** | ESLint + Prettier |
| **Testing** | Test your changes |
| **Documentation** | Update docs |

### Review Process

1. **Submit** pull request
2. **Review** by maintainers
3. **Feedback** and changes
4. **Merge** when approved

## Security Audits

### Audit Process

1. **Code review** — Manual review
2. **Testing** — Automated and manual
3. **Community** — Public review

### How to Audit

1. View the code on GitHub
2. Clone the repository
3. Review the code
4. Report issues
