# Feature Requests

We're always looking to improve ZRP. Your feedback helps us build better tools.

## How to Submit a Feature Request

### Option 1: Contact Form

1. Go to the [Contact page](/contact)
2. Subject: **"Feature Request: [Feature Name]"**
3. Describe:
   - What you want
   - Why it's useful
   - How it would work

### Option 2: X (Twitter)

1. Tag @ZRP_AI
2. Include: **"Feature Request:"**
3. Describe your idea
4. Why it's valuable

### Option 3: Discord

1. Join our [Discord](https://discord.gg/W4qS4xkbn)
2. Post in **#feature-requests**
3. Describe your idea
4. Discuss with the community

### Option 4: GitHub

1. Go to our [GitHub](https://github.com/LeonidasSpart/solanalaunchpad)
2. Open an issue
3. Label it: **"feature-request"**
4. Describe your idea

## What Makes a Good Feature Request

### Clear Description

| Component | What to Include |
|-----------|-----------------|
| **What** | Describe the feature |
| **Why** | Explain the use case |
| **How** | How it should work |
| **Who** | Who it helps |

### Examples

**Good Request:**
```

Feature: Token Analytics Dashboard

What: A dashboard showing real-time token metrics
Why: Helps users track their token's performance
How: Connect to Jupiter API for price and volume data
Who: All token creators

```

**Bad Request:**
```

Add more features

```

## What We Look For

### Prioritization Factors

| Factor | Why It Matters |
|--------|----------------|
| **User benefit** | Helps many users |
| **Implementation cost** | Time and complexity |
| **Platform fit** | Aligns with ZRP's goals |
| **Impact** | Improves the experience |

## Feature Request Process

### Step 1: Submit

1. Submit your request
2. Clear description
3. Provide examples

### Step 2: Review

1. Team reviews the request
2. Feasibility assessment
3. Priority evaluation

### Step 3: Update

1. You'll be informed
2. Timeline (if approved)
3. Progress updates

### Step 4: Release

1. Feature is built
2. Tested thoroughly
3. Released to users

## What We Won't Build

### Requests We Typically Decline

- ❌ Centralized features
- ❌ Custodial solutions
- ❌ Unrelated to token creation
- ❌ Highly specialized tools

### Why We Decline

- Doesn't align with mission
- Too complex for value
- Outside expertise
- Security concerns
