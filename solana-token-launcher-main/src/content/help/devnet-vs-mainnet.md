# Devnet vs Mainnet

ZRP supports both Solana Devnet and Mainnet. Understanding the difference helps you choose the right network for your needs.

## What is Devnet?

**Devnet** is a test network for Solana developers. It's designed for testing and experimentation with no real financial risk.

### Devnet Features

| Feature | Description |
|---------|-------------|
| **Free SOL** | Get test SOL from a faucet |
| **No Real Value** | Tokens have no real monetary value |
| **Testing** | Perfect for learning and experimenting |
| **Zero Risk** | No financial loss if you make mistakes |

### How to Get Devnet SOL

1. Visit a Solana faucet like [faucet.solana.com](https://faucet.solana.com)
2. Enter your wallet address
3. Request test SOL
4. Wait for confirmation

## What is Mainnet?

**Mainnet** is the live Solana network where tokens have real value. This is where you deploy production tokens.

### Mainnet Features

| Feature | Description |
|---------|-------------|
| **Real SOL** | SOL has real monetary value |
| **Real Tokens** | Tokens can be traded and have value |
| **Production** | For serious projects and launches |
| **Cost** | Transaction fees cost real SOL |

### Mainnet Costs

| Item | Cost |
|------|------|
| **Base Token Creation** | 0.15 SOL |
| **Revoke Mint Authority** | +0.15 SOL |
| **Revoke Freeze Authority** | +0.15 SOL |
| **Revoke Update Authority** | +0.15 SOL |
| **Total (All Features)** | 0.60 SOL |

## When to Use Each Network

### Use Devnet For:

- ✅ Learning how to create tokens
- ✅ Testing your token before launch
- ✅ Experimenting with settings
- ✅ No cost exploration

### Use Mainnet For:

- ✅ Launching a real project
- ✅ Creating tokens with actual value
- ✅ Trading on DEXs like Raydium
- ✅ Building a community around your token

## How to Switch Networks on ZRP

1. Look for the **Devnet/Mainnet** toggle in the header
2. Click to switch between networks
3. Your wallet will also need to be on the same network

## Important Note

> ⚠️ **Devnet and Mainnet are completely separate.** Tokens created on Devnet do not exist on Mainnet and cannot be transferred between networks.
